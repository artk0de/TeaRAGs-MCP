# TeaRAGs Brand & Identity

## 1. Project Identity

TeaRAGs is an open-source project built around a specific engineering philosophy:

**trajectory enrichment-aware retrieval for coding agents** — combining semantic retrieval with signals about how code evolves, changes, and interacts over time.

The project name **TeaRAGs**, its logo, and visual identity represent the original project and its evolving design philosophy.

Source code licensing is handled separately under the MIT License.

---

## 2. Use of the Name

You are welcome to:

- reference TeaRAGs in documentation, research, or comparisons
- build integrations and extensions
- discuss or cite the project in articles or tools
- build compatible tooling around the ecosystem

Please do not:

- publish modified versions under the same name
- use the TeaRAGs logo to imply official affiliation
- present derivative work as the original project

The goal is to keep the ecosystem clear and avoid confusion for users.

---

## 3. Forks and Derivative Projects

Forks are a natural and encouraged part of open-source development.

If you publish a modified version:

- use a distinct project name
- clearly mention that it is based on TeaRAGs
- include attribution to the original repository

This helps users understand lineage and preserves clarity in the ecosystem.

---

## 4. Attribution

A simple attribution is appreciated, for example:

> Based on TeaRAGs — trajectory enrichment-aware RAG for coding agents.

---

## 5. Why this policy exists

TeaRAGs is more than a codebase — it represents a specific approach to enriching retrieval with signals from code evolution, development history, and system structure.

This policy exists to:

- reduce confusion between original and modified projects
- preserve continuity of project identity
- keep the ecosystem open while respecting origin and context
