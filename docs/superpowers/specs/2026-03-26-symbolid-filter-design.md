# SymbolId Filter Design

**Date:** 2026-03-26 **Status:** Approved

## Problem

Users cannot filter search results by symbol name (class, method, function). The
`symbolId` field exists in Qdrant payload but has no dedicated filter parameter.

## Design

### SymbolId Format

Format is `ParentName.childName` (dot separator), generated by
`TreeSitterChunker.buildSymbolId()`:

| Chunk type         | Format                 | Example                        |
| ------------------ | ---------------------- | ------------------------------ |
| Class method       | `ClassName.methodName` | `ExploreFacade.semanticSearch` |
| Top-level function | `functionName`         | `createAppContext`             |
| Class/interface    | `ClassName`            | `VoyageError`                  |
| Markdown block     | arbitrary text         | `Anti-patterns`                |

Dot separator is used universally (no Ruby `#` distinction for instance vs class
methods — tracked in `tea-rags-mcp-q7s4`).

### Filter Behavior

Uses Qdrant `match.text` on the `symbolId` field with a text index. This gives
partial match:

- `"ExploreFacade.semanticSearch"` — exact match
- `"ExploreFacade"` — all methods of the class
- `"semanticSearch"` — method in any class
- `"Facade"` — any symbol containing "Facade"

### Schema Migration (v8)

Add text index on `symbolId` field:

- `ensureCurrentSchema()`: v8 migration block —
  `ensurePayloadIndex(collection, "symbolId", "text")`
- `initializeSchema()`: add `symbolId` text index for new collections
- Bump `CURRENT_SCHEMA_VERSION` from 7 to 8

### Filter Implementation

Static filter in `trajectory/static/filters.ts`:

```typescript
{
  param: "symbolId",
  description: "Filter by symbol ID...",
  type: "string",
  toCondition: (value: unknown) => ({
    must: [{ key: "symbolId", match: { text: value as string } }],
  }),
}
```
